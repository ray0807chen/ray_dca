# RAY DCA - 高中自主學習作品
網頁連結：<https://ray0807chen.github.io/ray_dca/>

這是一個美股投資回測網站，用於分析定期定額（DCA）與單筆投入（All-in）的歷史績效。
由 RAY 開發，作為高中自主學習成果展示。

## 學習動機與目標

定期定額投資（Dollar-Cost Averaging）是行為財務學中被廣泛討論的策略。其核心概念是透過固定頻率、固定金額的方式分批買入，降低單一時間點進場的擇時風險。本專題的出發點是：DCA 與 All-in 在不同時間區間、不同股票類型下，究竟哪一種策略的實際報酬更高？

為了回答這個問題，我使用真實歷史收盤價進行量化回測，而非依賴主觀判斷或模擬數據。

## 應用的經濟與財務概念

**年化報酬率（CAGR）**

複利報酬的標準化指標，計算方式為：

```
CAGR = (最終市值 / 總投入成本) ^ (1 / 年數) - 1
```

不同期間的績效必須轉換為年化才能公平比較，例如 3 年翻倍與 10 年翻倍的實際報酬差距懸殊。

**定期定額（DCA）的平均成本效應**

每月固定投入時，股價低時自動買到較多股數，高時買較少。長期下來，實際平均成本低於算術平均股價。本站的計算方式是以每月收盤價模擬買入，逐月累積持股數量。

**機會成本與 All-in 的比較**

All-in 將全部資金一次投入，若起始點恰好在高點，承受的回撤風險大；DCA 則犧牲部分「早期複利」換取分散時間的安全感。哪種策略更好，取決於市場環境與個人風險偏好，這也是本工具設計的出發點。

**股票卡片的漲跌幅**

由最新交易日收盤價與前一交易日收盤價計算，反映最近一個交易日的日報酬率，計算式為：

```
日報酬率 = (最新收盤 - 前日收盤) / 前日收盤 × 100%
```

## 學習反思

在建立這個工具的過程中，我意識到歷史回測存在倖存者偏差（Survivorship Bias）：納入回測的都是現今仍存活的知名公司，已倒閉或被購併的公司並未納入，這會使回測結果系統性偏高。

此外，過去績效不代表未來表現，這是所有量化回測都必須面對的根本限制。

## 網站功能

### 1. 股票定期定額計算

從 49 支美股及 ETF 中選擇一支，設定下列參數後即可查看回測結果：

- 投資起始日期
- 每月投資金額
- 每月投資日（1 至 28 日）
- 手續費率

計算結果包含：總投入成本、現值、總報酬率、年化報酬率（CAGR）、手續費總支出、損益金額，並附上投資組合歷史走勢圖。

### 2. 股票績效 PK

選擇兩支股票進行比較，支援兩種投資模式：

- All-in：設定一筆金額於起始日一次投入，追蹤後續市值變化。
- DCA：設定每月固定金額，逐月累積，計算實際持有市值。

比較期間可快速選擇 1、3、5、10、20 年。若某支股票的歷史數據不足所選年份，網站會跳出提示說明。結果以雙線圖呈現，並列出最終市值、總報酬率、年化報酬率的對比，自動判定勝出方。

### 3. 每日自動更新（真實數據）

股票卡片上顯示的漲跌幅，由最後兩個交易日的收盤價計算得出，非模擬數值。

股票歷史數據儲存於 `stock_data/` 資料夾，每個 JSON 檔案記錄每一交易日的收盤價。`update_data.py` 負責資料更新：若 JSON 已存在，僅補抓最新日期之後的資料；若不存在，則下載完整歷史數據。

### 4. 關於我頁面

靜態 HTML 頁面，填寫個人簡介、學習歷程、技能與聯絡資訊。

## 技術實現

| 項目 | 說明 |
|------|------|
| 前端 | HTML + Vanilla CSS + JavaScript（無框架依賴） |
| 樣式框架 | Bootstrap 5（排版）、自訂 CSS（設計語言） |
| 圖表 | Chart.js（投資績效走勢圖）、Highcharts（回測頁） |
| 數據來源 | Yahoo Finance，透過 yfinance（Python）抓取 |
| 自動化 | GitHub Actions 每天台灣時間 9:00 執行更新並部署 |
| 部署 | GitHub Pages（gh-pages 分支） |

## 自動更新流程

1. GitHub Actions 排程觸發（`daily-update.yml`）
2. 執行 `update_data.py`，以增量方式更新 `stock_data/*.json`
3. commit 並 push 至 main
4. 透過 `peaceiris/actions-gh-pages` 部署至 gh-pages

## 本地執行

```bash
# 安裝依賴
pip install yfinance pandas

# 更新數據（增量，已有資料只補最新幾天）
python update_data.py

# 用本地伺服器開啟（避免 CORS 問題）
python -m http.server 5500
```
